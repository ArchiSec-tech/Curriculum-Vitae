[ How I Test — Metodologia ]

Ecco i comandi e la metodologia che uso nei security assessment reali. Trasparenza totale per i recruiter tecnici.

1. Network Discovery & Port Scanning

Scansione iniziale rapida:
nmap -sS -T4 -p- --open target.com
Service detection e version scanning:
nmap -sV -sC -O -A -p 22,80,443,8080 target.com
Scansione UDP (servizi critici):
nmap -sU --top-ports 1000 target.com

2. Web Application Testing

Scan vulnerabilità web con Nikto:
nikto -h https://target.com -ssl -Format htm -output nikto_report.html
Test SSL/TLS configuration:
openssl s_client -connect target.com:443 -servername target.com
Check security headers:
curl -I -s https://target.com | grep -E "(X-Frame|Content-Security|Strict-Transport)"

3. Vulnerability Assessment

Directory enumeration:
dirb https://target.com /usr/share/dirb/wordlists/common.txt
Subdomain discovery:
amass enum -d target.com -o subdomains.txt
DNS enumeration:
dig target.com ANY +noall +answer

4. Documentation & Reporting

Report Structure che uso:

  • Executive Summary: Rischi principali in linguaggio business
  • Technical Findings: Vulnerabilità dettagliate con CVSS score
  • Evidence: Screenshot e command output come proof
  • Remediation: Steps concreti per il fix
  • Timeline: Priorità di intervento (Critical/High/Medium/Low)

Tools per reporting: Markdown → HTML/PDF, screenshot automatici con scrot/gnome-screenshot

Nota: Tutti i test vengono eseguiti solo su sistemi per cui ho autorizzazione esplicita. Uso sempre VPN e documento ogni attività per compliance e trasparenza.
